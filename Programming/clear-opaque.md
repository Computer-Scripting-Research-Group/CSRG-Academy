
# Clear and Opaque Preprocessing
At CSRG, we divide the concept of preprocessing into two categories.
## Clear Preprocessing
* Produces a Human Readable Text File
* Produces a Machine Readable Text File
* Expanded Code is Usually Preserved
## Opaque Preprocessing
* Does not Produce a Human Readable Text File
* Produces a Machine Readable Text File
* Used in C++ Macros and Templates
* Expanded Code is Usually Discarded After Compilation
